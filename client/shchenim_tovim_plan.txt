3. פיצ'רים — מה כל אחד דורש 
פיצ'ר 1: פרסום ועיון בבקשות
משתמש יכול לפרסם בקשה לעזרה ושכנים יכולים לצפות בה ולהגיב.
Frontend
* טופס יצירת בקשה: שדות — כותרת, תיאור, קטגוריה, דחיפות, מיקום (כתובת ידנית)
* אפשרות העלאת תמונה לבקשה (דרך Cloudinary)
* רשימת בקשות פעילות — כרטיסייה לכל בקשה עם פרטים קצרים
* עמוד פרטי בקשה — פרטים מלאים + כפתור 'אני אעזור'
* סינון לפי קטגוריה, דחיפות ומרחק
* חיפוש חופשי לפי טקסט
Backend
* POST /requests — יצירת בקשה חדשה
* GET /requests — שליפת כל הבקשות (עם פרמטרי query לסינון)
* GET /requests/:id — פרטי בקשה
* PATCH /requests/:id — עדכון סטטוס בקשה (פתוח/נסגר/נלקח)
* DELETE /requests/:id — מחיקת בקשה (יוצר או מנהל)
* Middleware לאימות בעלות על הבקשה
מסד נתונים
* טבלת requests: id, user_id, title, description, category, urgency, location_text, lat, lng, image_url, status, created_at


פיצ'ר 2: מפה אינטראקטיבית
תצוגה גיאוגרפית של בקשות פעילות באזור הקרוב.
Frontend
* שילוב Google Maps JavaScript API בדף
* סמנים על המפה לכל בקשה פעילה
* לחיצה על סמן — popup עם פרטי הבקשה + קישור לעמוד מלא
* זיהוי מיקום המשתמש (Geolocation API)
* סינון רדיוס — הצגת בקשות בטווח X ק״מ
Backend
* GET /requests/nearby?lat=&lng=&radius= — שליפת בקשות לפי מרחק
* חישוב מרחק בצד השרת עם נוסחת Haversine (ללא API חיצוני)
* שדות lat/lng שמורים בטבלת requests
מסד נתונים
* שדות lat, lng מסוג FLOAT בטבלת requests


פיצ'ר 3: צ'אט בזמן אמת
שיחה ישירה בין מי שפרסם בקשה לבין המתנדב שלקח אותה — בזמן אמת.
Frontend
* ממשק צ׳אט — חלון הודעות + שדה קלט + כפתור שליחה
* הודעות נטענות מהשרת בעת פתיחת השיחה
* האזנה לאירועי Socket.io לקבלת הודעות חדשות
* הצגת 'מתקליד...' (אופציונלי)
* רשימת שיחות פעילות של המשתמש
Backend
* GET /chats — כל השיחות של המשתמש
* GET /chats/:id/messages — הודעות שיחה ספציפית
* POST /chats — פתיחת שיחה חדשה (בין מבקש למתנדב)
* Socket.io event: 'send_message' — שמירה ב-DB + emit לחדר
* Socket.io event: 'join_chat' — הצטרפות לחדר שיחה
מסד נתונים
* טבלת chats: id, request_id, requester_id, helper_id, created_at
* טבלת messages: id, chat_id, sender_id, content, created_at


פיצ'ר 4: לוח מודעות קהילתי
ועד הבית מפרסם הודעות רשמיות לדיירים. תושבים יכולים לראות אותן בלבד.
Frontend
* רשימת הודעות ממוינות לפי תאריך
* כרטיסייה לכל הודעה: כותרת, תוכן, שם הפרסם, תאריך
* ממשק פרסום הודעה (גלוי רק לועד בית / מנהל אזור)
* עמוד פרטי הודעה
Backend
* GET /announcements — שליפת כל ההודעות
* POST /announcements — פרסום הודעה (תפקיד: board_manager או area_admin)
* DELETE /announcements/:id — מחיקת הודעה (מנהל בלבד)
* Middleware לבדיקת תפקיד לפני פרסום
מסד נתונים
* טבלת announcements: id, author_id, title, content, created_at

4. רכיבים משותפים — למנוע כפילויות
אלה הדברים שיותר מפיצ'ר אחד צריך אותם. יש להסכים עליהם ולכתוב פעם אחת.


Auth — אימות ותפקידים
* JWT middleware לשרת — פונקציה אחת שמאמתת token ומוסיפה req.user
* Role guard middleware — מקבל מערך תפקידים מורשים, מחזיר 403 אחרת
* Context/Hook בצד Client — useAuth() שמחזיר user, token, login(), logout()
* Protected Route component ב-React לעמודים שדורשים התחברות


API Layer בצד Client
* קובץ api.js מרכזי עם base URL ו-axios instance עם interceptor לשליחת token
* כל הפיצ'רים קוראים לפונקציות מהקובץ הזה ולא כותבים fetch ישירות


רכיב UploadImage
* רכיב React יחיד שמקבל קובץ ושולח ל-Cloudinary
* מחזיר URL של התמונה ששמורה בענן
* בשימוש בטופס בקשה ובפרופיל משתמש


מבנה DB משותף
* טבלת users: id, email, password_hash, name, role, building_id, avatar_url, created_at, is_blocked
* Enum לתפקידים: resident, board_manager, area_admin
* כל הטבלאות האחרות כוללות foreign key ל-users.id


Toast / Notification component
* רכיב הודעות מרכזי — הצלחה, שגיאה, אזהרה
* בשימוש בכל הפיצ'רים אחרי פעולות (שמירה, שגיאת רשת, וכד׳)


Socket.io חיבור מרכזי
* חיבור ל-socket נפתח פעם אחת ב-App.jsx (לא בכל קומפוננט מחדש)
* מועבר ל-context כדי שכל קומפוננט יוכל להשתמש בו

7. משימות נוספות — לחלוקה בין חברי הצוות
אלה משימות עצמאיות שאינן חלק מ-4 הפיצ'רים הראשיים. כל אחת מתאימה לאדם אחד.


אימות משתמשים
* הרשמה — טופס + שליחה ל-POST /auth/register + שמירת token
* התחברות — טופס + שליחה ל-POST /auth/login + עדכון AuthContext
* שמירת token ב-localStorage + שליחה אוטומטית בכל בקשה
* Logout — ניקוי localStorage + איפוס context


פרופיל משתמש-שחר
* עמוד פרופיל: הצגת שם, אימייל, תמונה, כמות בקשות שפרסמתי
* עריכת שם ותמונת פרופיל (Cloudinary)
* רשימת הבקשות שפרסמתי + שבהן עזרתי


דירוג משתמשים-שחר
* לאחר סגירת בקשה — אפשרות לדרג את המתנדב (1-5 כוכבים)
* שמירת הדירוג ב-DB
* הצגת ממוצע דירוג בכרטיסיית משתמש
* טבלת ratings: id, rated_user_id, rater_user_id, request_id, score, created_at


סגירת בקשה
* כפתור 'סמן כנפתרה' בעמוד פרטי הבקשה
* שינוי status ל-'closed' + trigger לפתיחת חלון דירוג
* הסרת הבקשה מרשימת הפעילות




הודעות Email / Push (אופציונלי — Bonus)
* שליחת email כשמישהו לוקח את הבקשה שלי
* שימוש ב-Nodemailer עם Gmail SMTP
* הגדרת template הודעה


עיצוב ו-UX-נגה
* רכיבי UI משותפים: Button, Input, Card, Badge, Modal
* Loading states ו-skeleton screens
* עמוד 404


Testing & Deployment
* כתיבת בדיקות ל-API עם Postman / Thunder Client
* הגדרת GitHub Actions — בדיקות אוטומטיות על PR
* Deploy Backend ל-Render
* Deploy Frontend ל-Vercel
* הגדרת משתני סביבה ב-Render ו-Vercel
